04
STAGE 04 Retrieval Engine no vector DB — deterministic CSV lookup
Entity extraction
Regex strips model strings (e.g. VMP5000) before extracting numeric alarm codes. Exact and prefix matching for site names. Skip-word list prevents turbine type false positives from generic words.
regexno NLPskip-words
3-level fallback search
Tries exact site + turbine match → drops turbine type → drops site → drops both. Each fallback level injects a CONFIDENCE NOTE into the LLM context so the model calibrates its answer accordingly.
deterministictransparentauditable
Keyword comment search
Stop-word filtered keyword matching across technician comments. Activated when structured lookup yields no results. Returns up to 20 most relevant records ranked by keyword hit.
keywordfallback20 records
05
STAGE 05 LLM Layer GPT-4.1 · temp 0.3 · 1024 tokens
Model: GPT-4.1
Temperature 0.3 and max 1,024 tokens with streaming enabled. Low temperature chosen to minimise hallucination in a safety-critical field operations context where incorrect guidance has real consequences.
OpenAIstreamingtemp 0.3
System prompt design
8 strict rules: data-only answers with no invented steps, (Source: Name) citation format, no raw CSV filenames exposed, Vestas PDF links always included, off-topic rejection, and mandatory follow-up suggestions phrased as technician queries.
8 rulesgroundedsafe
Response structure
Streaming answer → (Source: Manual / Technician Comments / Sites) citation tags → clickable Vestas PDF reference links → 3 contextual follow-up questions, each phrased in the voice of the technician asking.
citedstreamedfollow-ups
